1. 截止 2025.02，Dify 的 Web 后端仍使用同步调用，高并发下延迟明显，不建议在强实时任务中使用。

2. Tesseract OCR 需要让待识别文字周围保留几个像素的空白边框，否则识别效果会很差。据说 EasyOCR 识别数字效果更好，本人未验证，而是用 Tesseract 指定数字识别模式。  
   官方说明：[Tesseract — Borders（ImproveQuality）](https://tesseract-ocr.github.io/tessdoc/ImproveQuality.html#:~:text=lines%20are%20horizontal.-,Borders,-Missing%20borders)

3. **问题：** 遇到 UAC 弹窗申请管理员权限（但实际用不到）的流氓软件，如何防止弹窗打断自启动？  
   **解答：** 编辑注册表 `HKEY_CURRENT_USER\Software\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Layers`，在右侧新建字符串值，命名为程序全路径（如 `C:\Program Files\Everything\Everything.exe`），将数值数据设为 `RunAsInvoker`，即可抑制 UAC 并以当前用户身份运行。

4. 在 `pywinauto` 中，部分组件为 `Wrapper` 类型，属性不全，需先用 `hasattr()` 判断再访问。

5. 钉钉机器人发送群消息：官方 Python 示例未覆盖的坑见 [CSDN 说明](https://blog.csdn.net/mccgodlov/article/details/120759933)。

6. Qt 编写的软件用 pywinauto 等库较难解析控件结构；Python 的 Qt 库也不是干这个的。

7. `git pull` 后可能丢失可执行位（`+x`），脚本无法直接执行；用 `/bin/bash` 调用 Shell 脚本更稳妥。

8. C++ 与 Python 生成 bin 时默认行为不同：GCC 受内存对齐影响（可能 4/8 字节等），Python 不对齐，会导致 bin 不一致。

9. 接上条：C++ 的填充字节内容未定义，可能造成文件摘要不一致、按任何编码解码均报错等问题。

10. **Cursor Remote SSH 服务端升级失败时的处理**
    - 找到正在下载的 `wget` 进程，记下 PID（第二列）。

      ```bash
      ps ux | grep downloads.cursor.com
      ```

    - 用 PID 查出该 `wget` 进程的下载目录（工作目录）。

      ```bash
      readlink /proc/<PID>/cwd
      ```

    - 从 `wget` 命令行复制下载 URL（形如
      `https://downloads.cursor.com/production/<commit>/linux/x64/cursor-reh-linux-x64.tar.gz`），
      本机下载后上传到上一步查到的目录；保持 URL 末尾的文件名即可，不必改成 `wget -O` 的
      `cursor-server-<uuid>.tar.gz`。

    - 进入该下载目录后解压（`--strip-components=1` 用于去掉外层的 `vscode-reh-linux-x64` 目录）。

      ```bash
      cd <上一步 readlink 得到的目录>
      tar -xzf cursor-reh-linux-x64.tar.gz --strip-components=1
      ```

    - 重启 Cursor 即可。
